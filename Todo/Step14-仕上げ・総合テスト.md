# Step 14: 仕上げ・総合テスト

## 概要

全機能の総合テスト、不具合修正、ドキュメント更新を行う。

**参照**: [動作確認手順書](../docs/動作確認手順書.md)、[環境構築手順書](../docs/環境構築手順書.md)

## 状態

- [ ] 未実装
- [ ] 実装中
- [ ] 完了

**優先度**: 高

## 実装内容

### 1. 総合テスト

| 項目 | 内容 |
|------|------|
| 管理者フロー | ログイン→ユーザー登録→年度・学期→科目→提出物→採点 |
| 教員フロー | ログイン→科目確認→提出物作成→提出状況→採点 |
| 学生フロー | ログイン→課題一覧→提出→成績確認 |
| 管理者学生フロー | ログイン→担当科目→提出状況更新 |
| 権限テスト | ロールごとのアクセス制御確認 |
| エッジケース | バリデーション、エラーハンドリング |

### 2. 調整・修正

| 項目 | 内容 |
|------|------|
| UI/UX | レスポンシブ、アクセシビリティ、一貫性 |
| パフォーマンス | 遅い画面の最適化 |
| セキュリティ | XSS、CSRF、認証の抜け漏れ確認 |

### 3. ドキュメント更新

| 項目 | 内容 |
|------|------|
| 開発時の注意事項 | 新機能の設定・注意点を追記 |
| 動作確認手順書 | フロントエンド確認手順の追加 |
| README | セットアップ・起動方法の最終確認 |

### 4. テストスクリプト拡張（任意）

| 項目 | 内容 |
|------|------|
| E2Eテスト | Playwright 等でのブラウザ自動テスト |
| フロントエンド単体テスト | Vitest 等 |
